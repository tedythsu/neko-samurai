## 一、世界觀與故事背景

戰國時代末期，妖怪大軍（妖獣軍）入侵貓咪王國「猫ノ国」。玩家扮演流浪貓武士，孤身深入各地戰場驅除妖怪，蒐集「武魂」以強化自身，最終挑戰妖王降臨。

**風格定調：** 日式和風 × 8-bit 現代精緻像素（32×32 sprite，顯示時 2x 放大）× 可愛萌系色調

## 二、核心遊戲機制

### 2.1 遊戲流程

```
主選單 → 選角色 + 選地圖 → 生存局（15分鐘）
  → Wave Boss 排程（見下表）
  → 15:00：「魔王降臨」最終Boss（取代一般生成）
  → 結算畫面：獲得武魂
  → 回主選單（武魂可消費於強化樹）
```

**Wave Boss 排程（竹林村範例）：**

| 時間 | 事件 |
|------|------|
| 01:00 | Wave Boss 1：大鬼兵將（HP×1）|
| 03:00 | Wave Boss 2：大鬼兵將×2（HP×1.2）|
| 05:00 | Wave Boss 3：大鬼兵將×3（HP×1.5）|
| 08:00 | Wave Boss 4：大鬼兵將 強化型（HP×2，追蹤模式）|
| 11:00 | Wave Boss 5：妖狐前鋒（HP×2.5，衝刺攻擊）|
| 15:00 | 最終Boss：竹林妖狐（HP×5，多技能模式）|

- 每次 Wave Boss 死亡後，敵人生成速率 +15%（最高上限：基礎速率 ×2.5）
- 達到 150 敵人上限時，生成暫停但速率累計保留；擊殺後立即補充至上限
- Wave Boss 同時只存在 1 隻，未擊殺時不進入下一 Wave
- 最終 Boss 出現後停止一般敵人生成，集中戰鬥

**基礎無敵幀時間：** 受擊後 0.5 秒無敵（雪姬：0.5 × 1.5 = 0.75 秒），同時觸發 Knockback 效果，雙重保護玩家不被瞬間疊傷

### 2.2 角色系統（初版 3 隻）

| 貓咪 | 品種 | 專屬被動 | 初始武器 |
|------|------|----------|----------|
| 三毛丸 | 三毛貓 | 金幣獲取 +20% | 苦無（多方向飛刀）|
| 黑影 | 黑貓 | 暴擊率 +15% | 太刀斬擊（近距弧形）|
| 雪姬 | 白貓 | 受傷後無敵時間 +50% | 扇形冰符（冰凍效果）|

### 2.3 武器系統（初版 6 種）

| 武器 | 類型 | 描述 |
|------|------|------|
| 苦無投擲 | 投射物 | 向最近敵人投出飛刀 |
| 太刀斬擊 | 近距弧形 | 360°分段斬擊 |
| 火炎符 | 前方扇形 | 持續灼燒傷害 |
| 召喚式神 | 環繞衛星 | 繞玩家旋轉傷害 |
| 鎖鏈鐮刀 | 穿透線 | 穿越敵人的直線攻擊 |
| 扇形衝擊波 | 震波 | 向外擴散的環形衝擊 |

**武器成長：** 每種武器升至 Lv.5，合成需要：武器達 Lv.5 + 持有對應被動道具。合成時**僅消耗武器槽位**（武器升級為終極形態），**被動道具保留**並將其效果整合進終極武器的額外屬性中（例：疾風符 → 千本苦無附帶 +10% 移速加成）。終極武器最高 Lv.3，不可逆。

**合成配方：**
| 武器（Lv.5）| 所需道具 | 終極武器 | 效果 | 道具整合效果 |
|------------|----------|----------|------|------------|
| 太刀斬擊 | 鋼鐵護符 | 滅妖太刀 | 超大範圍 360° 斬擊，每 2 秒觸發 | 附帶 +15% 防禦力 |
| 苦無投擲 | 疾風符 | 千本苦無 | 向所有敵人同時投出，爆炸濺射 | 附帶 +10% 移速 |
| 扇形衝擊波 | 雷神符 | 雷鳴衝天波 | 全場持續電擊傷害，2 秒 tick | 附帶 +10% 冷卻縮減 |

### 2.4 升級系統

每次升等從 3 張牌中選一張：
- 武器升級（既有武器 +1 Lv）
- 新武器（未持有的武器）
- 道具（被動強化，如 +HP、+移速、+暴擊）

稀有度與基礎機率：

| 稀有度 | 顏色 | 基礎機率 | 幸運值每+10 |
|--------|------|----------|-------------|
| 普通 | 灰 | 70% | -5%（下限 30%）|
| 稀有 | 藍 | 25% | +3%（上限 50%）|
| 史詩 | 紫 | 5% | +2%（上限 20%）|

幸運值來源：角色被動、道具、武魂強化樹。幸運值每 +10 套用一次上表調整。

### 4.3 擴充性設計

- **資料驅動：** 所有角色、武器、敵人、地圖設定集中在 `data/` 目錄，新增內容不需修改核心邏輯

- **Electron 預留：** `electron/main.js` 已規劃，存檔系統設計支援 LocalStorage → fs 切換
- **音訊系統：** Phaser 3 Sound Manager，支援 BGM 淡入淡出（500ms）+ 音效層疊（最多 8 通道）

### 4.4 物理與碰撞

- **引擎：** Phaser 3 Arcade Physics（輕量，適合大量精靈）
- **玩家碰撞體：** 16×16 圓形（sprite 中下部）
- **敵人碰撞體：** 14×14 圓形
- **武器判定：** Overlap（非實體碰撞），命中後依武器穿透值決定是否繼續
- **Knockback（受擊推力）：** 玩家受傷時，以玩家為圓心、半徑 48px 內的敵人被向外推開（推力 120px/s，持續 0.2 秒），提供喘息空間並增強動作手感
- **效能目標：** 60 FPS，最多同時 **300 個敵人**（Object Pool 管理）
  - 距玩家 400px 以外的敵人停用 Arcade Physics 更新，僅以向量計算簡單追蹤，降低物理運算負擔
  - 超過 300 隻上限時暫停新生成；擊殺後立即補充

### 4.5 輸入對應

| 動作 | 鍵盤 | 手把（預留）|
|------|------|------------|
| 移動 | WASD / 方向鍵 | 左搖桿 |
| 確認（UI）| Enter / Space | A 鍵 |
| 取消（UI）| Escape | B 鍵 |
| 暫停 | P / Escape | Start |
| 攻擊 | 全自動 | 全自動 |

### 4.6 存檔格式

```js
// LocalStorage key: 'neko_meta_v1'
{
  version: 1,                          // 升版時用於遷移
  souls: 1240,                         // 累積武魂
  nodes: { atk: 2, hp: 3, spd: 1 },  // 各節點等級
  unlocked: ['kuroka', 'shrine'],      // 已解鎖角色/地圖 id
  stats: { totalRuns: 12, bestTime: 847, totalKills: 3421 }
}
```

版本遷移：讀取時若 `version` 不匹配，執行 migrate 函式補全缺失欄位，確保向前相容。

**暫停選單行為：** 遊戲中按 P / Escape 觸發 PauseScene overlay，遊戲時間暫停。提供：繼續、重新開始、返回主選單三個選項。返回主選單前確認對話框（本局進度不保存）。

## 五、UI/UX 設計

### 5.1 視覺風格

- **色調：** 深夜藍 / 墨黑背景 + 金黃、朱紅、白色文字
- **字體：** 等寬像素字體（Courier New fallback，正式版替換像素字型）

### 5.2 主要畫面

1. **主選單** — 夜晚場景、月亮、鳥居剪影、星空、主選單按鈕
2. **角色選擇** — 3 隻貓咪卡片 + 屬性說明 + 被動特性
3. **地圖選擇** — 地圖縮圖 + 難度標示 + 解鎖狀態
4. **遊戲 HUD** — 頂部：血條 + 計時器 + 等級 / 底部：武器槽 / 底邊：經驗條
5. **升級選牌** — 半透明遮罩 overlay，3 張卡片，稀有度顏色區分
6. **結算畫面** — 生存時間、擊殺數、武魂獲取、武器成長
7. **武魂強化樹** — 5×3 節點格，解鎖/未解鎖/滿級三種狀態

### 5.3 Visual Juicing（視覺加料規則）

所有視覺回饋效果為**必要實作**，不可省略：

| 事件 | 視覺效果 |
|------|----------|
| 普通敵人死亡 | 爆出 4-6 片「櫻花瓣」粒子，向外擴散後淡出（0.4 秒）|
| 妖怪系敵人死亡 | 爆出藍紫「魂火」粒子（6-8 顆），輕微上飄後消散 |
| Boss 死亡 | 大爆炸：30 顆混合粒子（櫻花 + 魂火）+ 閃光 + 鏡頭震動 0.3 秒 |
| 玩家受傷 | 紅色閃光遮罩（0.15 秒）+ Knockback + 角色閃爍（無敵期間）|
| 升等 | 金色光環從角色擴散 + 音效 |
| 武器命中 | 小型撞擊火花粒子（2-3 顆）|
| 武器合成 | 全螢幕金色粒子淋浴 + 捲軸展開動畫 |

粒子系統：使用 Phaser 3 `ParticleEmitter`，每種效果為獨立 emitter 設定，預先建立 pool 避免 GC。

### 5.4 升級選牌 UI 設計

走**捲軸展開**風格，而非純卡片：
- 背景：半透明黑幕 + 全螢幕模糊
- 三張「和紙捲軸」從上方展開（逐張，間隔 80ms），帶展開動畫（tween）
- 捲軸背景材質：米黃色 + 水墨紋理（程式繪製或貼圖）
- 稀有度以捲軸邊框印章顏色區分：灰印（普通）/ 青印（稀有）/ 紫印（史詩）
- 滑鼠 hover 時捲軸輕微放大（scale 1.05）
- 選擇後：未選中的兩張捲起收回，選中的捲軸向上飛出螢幕

### 5.5 輸入支援

- **鍵盤：** WASD / 方向鍵移動；Enter / Space 確認；Escape / P 暫停
- **滑鼠：** UI 操作（點擊選牌、強化樹、選單）
- **攻擊：** 完全自動，無需玩家操作
- **未來：** 手把支援（Phaser 3 Gamepad API 預留，按鍵對應見 4.5）

### 5.6 音效規格

- BGM：每張地圖 1 首循環（`.ogg` + `.mp3` fallback），淡入淡出 500ms
- 音效通道：最多 8 個同時播放，優先級：Boss 音效 > 武器音效 > 環境音
- UI 設定：音樂音量 / 音效音量兩個獨立滑桿，存入 LocalStorage

## 七、檔案命名與資源規範

- Sprite 命名：`{name}_{action}_{frame}.png`（例：`mikenomaru_idle_01.png`）
- 像素解析度：32×32（角色）/ 16×16（小道具）/ 48×48（Boss）
- 色板：每個角色限制 16 色以內，保持像素風統一感
- 音效格式：`.ogg`（主要）+ `.mp3`（fallback）
- Sprite 動畫幀數：

| 動作 | 幀數 | 備註 |
|------|------|------|
| idle | 4 幀 | 呼吸循環 |
| walk | 6 幀 | 4方向共用 |
| attack | 3 幀 | 攻擊揮砍 |
| hurt | 2 幀 | 受傷閃爍 |
| death | 5 幀 | 倒地消失 |

- Sprite Sheet 格式：單一橫向 PNG atlas（例：`mikenomaru.png` + `mikenomaru.json`），使用 Phaser 3 `this.load.atlas()`
- 視口縮放：遊戲解析度固定 480×270（內部），以 `scale.autoCenter` 等比縮放至視窗大小，保持像素整數倍，支援 HiDPI 顯示器
