# 猫の侍伝（Neko no Samurai-den）設計文件

**日期：** 2026-03-18
**專案路徑：** `/workspace/neko-samurai/`
**類型：** Roguelike 生存動作遊戲（Vampire Survivors 風格）
**平台：** 瀏覽器優先，預留 Electron 桌面版 / Steam 擴充

---

## 一、世界觀與故事背景

戰國時代末期，妖怪大軍（妖獣軍）入侵貓咪王國「猫ノ国」。玩家扮演流浪貓武士，孤身深入各地戰場驅除妖怪，蒐集「武魂」以強化自身，最終挑戰妖王降臨。

**風格定調：** 日式和風 × 8-bit 現代精緻像素（32×32 sprite，顯示時 2x 放大）× 可愛萌系色調

---

## 二、核心遊戲機制

### 2.1 遊戲流程

```
主選單 → 選角色 + 選地圖 → 生存局（15分鐘）
  → 每60秒出現Wave Boss → 擊殺後強化浪潮
  → 15分鐘：「魔王降臨」最終Boss
  → 結算畫面：獲得武魂
  → 回主選單（武魂可消費於強化樹）
```

### 2.2 角色系統（初版 3 隻）

| 貓咪 | 品種 | 專屬被動 | 初始武器 |
|------|------|----------|----------|
| 三毛丸 | 三毛貓 | 金幣獲取 +20% | 苦無（多方向飛刀）|
| 黑影 | 黑貓 | 暴擊率 +15% | 太刀斬擊（近距弧形）|
| 雪姬 | 白貓 | 受傷後無敵時間 +50% | 扇形冰符（冰凍效果）|

### 2.3 武器系統（初版 6 種）

| 武器 | 類型 | 描述 |
|------|------|------|
| 苦無投擲 | 投射物 | 向最近敵人投出飛刀 |
| 太刀斬擊 | 近距弧形 | 360°分段斬擊 |
| 火炎符 | 前方扇形 | 持續灼燒傷害 |
| 召喚式神 | 環繞衛星 | 繞玩家旋轉傷害 |
| 鎖鏈鐮刀 | 穿透線 | 穿越敵人的直線攻擊 |
| 扇形衝擊波 | 震波 | 向外擴散的環形衝擊 |

**武器成長：** 每種武器升至 Lv.5，特定武器 + 道具 = 合成終極武器（3種合成配方）

**合成範例：**
- 太刀斬擊 Lv.5 + 鋼鐵護符 → 「滅妖太刀」（超大範圍斬擊）
- 苦無投擲 Lv.5 + 疾風符 → 「千本苦無」（全方向爆炸）
- 扇形衝擊波 Lv.5 + 雷神符 → 「雷鳴衝天波」（持續場地傷害）

### 2.4 升級系統

每次升等從 3 張牌中選一張：
- 武器升級（既有武器 +1 Lv）
- 新武器（未持有的武器）
- 道具（被動強化，如 +HP、+移速、+暴擊）

稀有度：普通（灰）/ 稀有（藍）/ 史詩（紫），機率受幸運值影響。

### 2.5 元魂成長系統（跨局永久強化）

結算後獲得「武魂」，在主選單強化樹中消費：

**節點（15個）：**
- 攻擊力 / 最大HP / 移動速度 / 幸運 / 拾取距離
- 冷卻縮減 / 暴擊率 / 防禦力 / 金幣獲取+ / 經驗值+
- 解鎖角色（黑影、雪姬）/ 解鎖地圖（鬼門神社、魔王城）
- 隱藏節點 ×3（後續版本擴充）

## 四、技術架構

### 4.1 技術選型

| 技術 | 用途 |
|------|------|
| Phaser 3 | 遊戲引擎（精靈、物理、輸入、場景管理）|
| Vite | 開發伺服器 + 打包 |
| LocalStorage | 武魂存檔（瀏覽器版）|
| Electron（預留）| 桌面版 / Steam 打包 |
| Capacitor（預留）| iOS / Android 打包 |

### 4.2 專案結構

```
neko-samurai/
├── index.html
├── package.json
├── vite.config.js
├── electron/

│   ├── scenes/

│   │   ├── MapSelectScene.js  # 地圖選擇

│   │   ├── UpgradeScene.js   # 升級選牌 overlay
│   │   └── ResultScene.js    # 結算畫面

│   │   ├── Enemy.js          # 敵人基底類別

            └── 2026-03-18-neko-samurai-design.md
```

### 4.3 擴充性設計

- **資料驅動：** 所有角色、武器、敵人、地圖設定集中在 `data/` 目錄，新增內容不需修改核心邏輯
- **Scene 解耦：** Steam 成就 / 雲存檔 API 只需在對應 Scene 插入呼叫點
- **Electron 預留：** `electron/main.js` 已規劃，存檔系統設計支援 LocalStorage → fs 切換
- **音訊系統：** Phaser 3 Sound Manager，支援 BGM 淡入淡出 + 音效層疊

## 五、UI/UX 設計

### 5.1 視覺風格

- **色調：** 深夜藍 / 墨黑背景 + 金黃、朱紅、白色文字
- **字體：** 等寬像素字體（Courier New fallback，正式版替換像素字型）
- **像素尺寸：** Sprite 32×32，顯示 2x 縮放（64×64）

### 5.2 主要畫面

1. **主選單** — 夜晚場景、月亮、鳥居剪影、星空、主選單按鈕
2. **角色選擇** — 3 隻貓咪卡片 + 屬性說明 + 被動特性
3. **地圖選擇** — 地圖縮圖 + 難度標示 + 解鎖狀態
4. **遊戲 HUD** — 頂部：血條 + 計時器 + 等級 / 底部：武器槽 / 底邊：經驗條
5. **升級選牌** — 半透明遮罩 overlay，3 張卡片，稀有度顏色區分
6. **結算畫面** — 生存時間、擊殺數、武魂獲取、武器成長
7. **武魂強化樹** — 5×3 節點格，解鎖/未解鎖/滿級三種狀態

### 5.3 輸入支援

- **鍵盤：** WASD / 方向鍵移動（瀏覽器 + 桌面版）
- **滑鼠：** UI 操作
- **未來：** 手把支援（Phaser 3 Gamepad API 預留）

## 七、檔案命名與資源規範

- Sprite 命名：`{name}_{action}_{frame}.png`（例：`mikenomaru_idle_01.png`）
- 像素解析度：32×32（角色）/ 16×16（小道具）/ 48×48（Boss）
- 色板：每個角色限制 16 色以內，保持像素風統一感
- 音效格式：`.ogg`（主要）+ `.mp3`（fallback）
